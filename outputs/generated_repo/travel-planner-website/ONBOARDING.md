# ONBOARDING.md
## Welcome to the Travel Planner Website Project
The Travel Planner Website is an open-source project aimed at creating a comprehensive online platform for travelers to plan and book their trips. The project involves developing a range of features, including destination guides, itinerary builders, booking systems, user reviews, and ratings.

### Project Overview
Our project is focused on delivering a user-friendly and robust travel planning experience. We are currently working on implementing the following key features:
* Destination Guides: providing users with information about different travel destinations, including attractions, restaurants, and accommodations.
* Itinerary Builder: allowing users to create and customize their travel itineraries, including booking accommodations and activities.
* User Reviews and Ratings: enabling users to leave reviews and ratings for destinations, accommodations, and activities.
* Continuous Integration and Deployment (CI/CD) Pipeline: automating the testing, building, and deployment of the travel planner website.

### Getting Started
To get started with the project, follow these steps:

1. **Familiarize yourself with the project structure**: Our project is organized into the following main directories:
	* `src`: contains the source code for the website.
	* `tests`: contains the unit tests and integration tests for the website.
	* `docs`: contains documentation for the project.
2. **Read the key files**: Start by reading the following files:
	* `README.md`: provides an overview of the project and its goals.
	* `CONTRIBUTING.md`: outlines the guidelines for contributing to the project.
	* `DESTINATION_GUIDES.md`: describes the implementation of the destination guides feature.
	* `ITINERARY_BUILDER.md`: describes the implementation of the itinerary builder feature.
3. **Explore the codebase**: Browse through the `src` directory to get a sense of the code organization and structure.
4. **Check the issue tracker**: Visit the [issue tracker](https://github.com/TravelPlannerWebsite/issues) to see the current tasks and bugs that need to be addressed.

### Picking a Task
To pick a task, follow these steps:

1. **Check the issue tracker**: Browse through the open issues and find one that interests you.
2. **Filter by labels**: Use the labels to filter issues by type (e.g., bug, feature, documentation).
3. **Sort by priority**: Sort issues by priority to focus on the most important tasks.
4. **Claim an issue**: Once you've found an issue you'd like to work on, comment on the issue to claim it and let others know you're working on it.

### Asking Questions
If you have any questions or need help with a task, don't hesitate to ask. You can:

1. **Comment on an issue**: If you have a question related to a specific issue, comment on the issue.
2. **Open a new issue**: If you have a general question or need help with something that's not related to a specific issue, open a new issue with a descriptive title.
3. **Join our community chat**: We have a community chat where you can ask questions and get help from other contributors.

### Additional Resources
* [Project Wiki](https://github.com/TravelPlannerWebsite/wiki): contains additional documentation and resources for the project.
* [Code of Conduct](https://github.com/TravelPlannerWebsite/CODE_OF_CONDUCT.md): outlines the expected behavior for contributors.

By following these steps, you'll be well on your way to contributing to the Travel Planner Website project. Thank you for your interest in our project, and we look forward to seeing your contributions!
