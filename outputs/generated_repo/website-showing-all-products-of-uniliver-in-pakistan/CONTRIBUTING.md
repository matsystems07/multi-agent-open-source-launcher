# CONTRIBUTING.md
## Introduction
Thank you for considering contributing to the Website Showing All Products Of Unilever In Pakistan project. This project aims to provide a comprehensive platform for users to explore and discover Unilever products available in Pakistan. We appreciate your interest in contributing to this project and are excited to have you on board.

## Getting Started
To contribute to this project, follow these steps:

1. **Fork the repository**: Navigate to the project's GitHub repository and click the "Fork" button in the top-right corner. This will create a copy of the repository in your own GitHub account.
2. **Clone the repository**: Clone the forked repository to your local machine using the command `git clone https://github.com/your-username/website-showing-all-products-of-unilever-in-pakistan.git`.
3. **Create a new branch**: Create a new branch for your feature or bug fix using the command `git checkout -b feature/your-feature-name`. Our branch naming convention is as follows:
	* `feature/` for new features
	* `fix/` for bug fixes
	* `docs/` for documentation changes
	* `refactor/` for code refactoring
	* `test/` for test additions or changes
	* `chore/` for miscellaneous changes

## Pull Request Expectations
When submitting a pull request, please ensure that:

1. **Your code is formatted according to our code style expectations**: We follow the standard JavaScript and HTML/CSS coding conventions. Please use a consistent naming convention, indent your code with 4 spaces, and keep your lines under 80 characters.
2. **Your code includes relevant tests**: We use Jest for testing. Please include tests for your new feature or bug fix.
3. **Your code is documented**: Please include relevant comments and documentation for your code.
4. **Your pull request is descriptive**: Please include a clear and concise description of your changes, including any relevant screenshots or examples.

## Code Style Expectations
We follow the standard JavaScript and HTML/CSS coding conventions. Please ensure that your code adheres to the following guidelines:

1. **Indentation**: Use 4 spaces for indentation.
2. **Line length**: Keep your lines under 80 characters.
3. **Naming convention**: Use camelCase for variable and function names.
4. **Comments**: Use JSDoc-style comments for functions and variables.
5. **Code organization**: Keep your code organized and modular, with each module having a single responsibility.

## Issue Reporting Guidance
When reporting an issue, please include the following information:

1. **Clear description**: Please include a clear and concise description of the issue, including any relevant screenshots or examples.
2. **Steps to reproduce**: Please include the steps to reproduce the issue, including any relevant code or configuration.
3. **Expected behavior**: Please include the expected behavior, including any relevant screenshots or examples.
4. **Actual behavior**: Please include the actual behavior, including any relevant screenshots or examples.

## How to Contribute
To contribute to the project, please follow these steps:

1. **Choose an issue**: Browse our issue tracker and choose an issue that you would like to work on.
2. **Create a new branch**: Create a new branch for your feature or bug fix using the command `git checkout -b feature/your-feature-name`.
3. **Make your changes**: Make your changes and commit them using the command `git commit -m "your-commit-message"`.
4. **Push your changes**: Push your changes to your forked repository using the command `git push origin feature/your-feature-name`.
5. **Submit a pull request**: Submit a pull request to the main repository, including a clear and concise description of your changes.
